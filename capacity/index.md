# 性能指标

URocketMQ核心指标TPS（每秒处理消息数）计算规则如下：

每秒生产与消费消息数量之和，按照消息KB值计算。其中：
- 4.x 版本消息平均大小以 1KB 为基准，不足 1KB 按 1KB 算。
示例：
1. 消息平均大小1KB，每秒同时生产与消费的消息分别为5000TPS与5000，则总TPS为1*(5000+5000)=10000
2. 消息平均大小2KB，每秒同时生产与消费的消息分别为5000TPS与5000，则总TPS为2*(5000+5000)=20000

- 5.x 版本消息平均大小以 4KB 为基准，不足 4KB 按 4KB 算。
示例：
1. 消息平均大小4KB，每秒同时生产与消费的消息分别为5000TPS与5000，则总TPS为1*(5000+5000)=10000
2. 消息平均大小8KB，每秒同时生产与消费的消息分别为5000TPS与5000，则总TPS为2*(5000+5000)=20000

当前提供的规格如下：

- 4.x 版本：2万TPS、5万TPS、10万TPS、20万TPS
- 5.x 版本：1万TPS、2万TPS、5万TPS、10万TPS、20万TPS

如有更高规格需求，可联系技术支持。
